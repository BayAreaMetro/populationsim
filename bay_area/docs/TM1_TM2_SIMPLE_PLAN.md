## Background

### The Situation

We have two branches that have diverged:

| Branch | What it does today |
|--------|--------------------|
| `master` | Runs TM1 (default) or legacy TM2 synthesis via `run_populationsim.bat` |
| `tm2` | Runs TM2 synthesis only, modernized pipeline (`tm2_pipeline.py`) |

**Goal:** Make the `tm2` branch also run TM1 synthesis by adding `--model_type TM1` support to `tm2_pipeline.py`.

---

### How the Two Pipelines Differ

| Step | TM1 | TM2 |
|------|-----|-----|
| Controls source | Copied in from `travel-model-one` R scripts (`TAZ1454 YEAR Popsim Vars.csv`) | Generated by `create_baseyear_controls.py` (Census API) |
| Controls prep | `add_hhgq_combined_controls.py --model_type TM1` | Same script, TM2 path |
| `configs_` dir | `hh_gq/configs_TM1/` (TAZ geography) | `hh_gq/configs_TM2/` (MAZ geography) |
| Geo crosswalk | `geo_cross_walk_tm1.csv` (TAZ‚ÜíPUMA‚ÜíCOUNTY) | `geo_cross_walk_tm2.csv` (MAZ‚ÜíTAZ‚ÜíPUMA‚ÜíCOUNTY) |
| PUMS vintage | 2017‚Äď2021 (2010-vintage PUMA codes) | 2019‚Äď2023 (2020-vintage PUMA codes) |
| Income field | `hh_income_2000` | `hh_income_2010` |
| Postprocess | `postprocess_recode.py --model_type TM1` | `postprocess_recode.py --model_type TM2` |
| Summaries | TAZ-level scripts only | MAZ + TAZ scripts |

---

### What Is Already Done (no changes needed)

- `postprocess_recode.py` on `tm2` already handles both `--model_type TM1` and `TM2`
