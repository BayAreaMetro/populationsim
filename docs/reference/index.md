# Detailed Reference Documentation

Comprehensive technical documentation for advanced users and developers.

## Complete Technical Guides

These are extensive, detailed documents that cover every aspect of the pipeline. They're intended for:
- Developers modifying the pipeline
- Advanced users needing complete technical specifications
- Troubleshooting complex issues

### Available Documents

#### [TM2 Full Reference](TM2_FULL_REFERENCE.html)
Complete technical reference covering all aspects of the TM2 PopulationSim pipeline.
- **Length**: ~460 lines
- **Topics**: All components, detailed specifications, advanced configuration

#### [Detailed Output Guide](DETAILED_OUTPUT_GUIDE.html)
Comprehensive documentation of all output files, formats, and validation procedures.
- **Length**: ~550 lines
- **Topics**: File specifications, field definitions, quality metrics

#### [Detailed Synthesis Guide](DETAILED_SYNTHESIS_GUIDE.html)
In-depth explanation of the PopulationSim algorithm and configuration options.
- **Length**: ~530 lines
- **Topics**: IPF algorithm, integerization, convergence, advanced settings

#### [Detailed Input Data Guide](DETAILED_INPUT_DATA_GUIDE.html)
Complete documentation of all input data sources, preparation, and validation.
- **Length**: ~470 lines
- **Topics**: PUMS data, geographic files, control sources, data quality

#### [Detailed Crosswalk Guide](DETAILED_CROSSWALK_GUIDE.html)
Comprehensive guide to geographic crosswalk creation and validation.
- **Length**: ~400 lines
- **Topics**: Zone relationships, conflict resolution, validation procedures

#### [Detailed Control Generation Guide](DETAILED_CONTROL_GENERATION_GUIDE.html)
Complete documentation of marginal control generation from Census sources.
- **Length**: ~360 lines
- **Topics**: ACS tables, data extraction, aggregation, balancing

## When to Use These Documents

**Use the focused guides** in the [Guides](../guides/) section if you:
- Are learning the pipeline for the first time
- Need quick reference for a specific topic
- Want step-by-step instructions

**Use these detailed references** if you:
- Need complete technical specifications
- Are modifying the pipeline code
- Are troubleshooting complex issues
- Need to understand all configuration options
- Are documenting or auditing the methodology
